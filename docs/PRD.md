# PRD — AI Beat Challenge

## 1) Background
### Problem
- 기존 차트 분석 서비스는 반복 사용 동기가 약하고, 결과 확인 루프가 약함.
- 모바일 사용 패턴(짧고 빈번한 세션)에 맞는 구조가 부족함.
- AI 예측 서비스는 규제/도박성 오해 리스크가 존재함.

### Market Opportunity
- 짧은 세션 기반 게임형 서비스는 리텐션이 높음.
- “AI와 대결” 프레임은 학습 + 경쟁 동시 충족.
- 모바일 우선 UX에 최적.

### Competitor Landscape (요약)
- 차트 분석 앱: 정보 제공 중심 → 재미/루프 약함.
- 트레이딩 시그널 앱: 신뢰/규제 리스크.
- 캐주얼 예측 게임: 데이터/신뢰성 약함.

## 2) Goals
### Business Goals
- 일일 활성 사용자(DAU) 확보
- 반복 플레이 유도 (짧은 세션)
- 규제 리스크 최소화

### User Goals
- AI와 대결하며 실력 향상 느낌
- 짧은 시간에 승부 결과 확인
- 보상/연승으로 동기 부여

### Success Metrics (KPI)
- D1 retention ≥ 25%
- D7 retention ≥ 10%
- 평균 세션 길이 3~6분
- 1인당 일 평균 플레이 2회 이상

## 3) Features
### Must Have
- 모드 선택 (FLASH/SPEED)
- 3라운드 예측 제출 (UP/DOWN)
- AI 예측 (세션당 1회)
- 결과 판정 (캔들 close 기준)
- 포인트 지급 (현금화 불가)
- 기본 전적 기록
- 면책/학습 목적 안내

### Nice to Have
- 중간 결과 알림 (라운드별)
- 연승 보너스
- 데일리 미션
- 시즌 배지

### Out of Scope
- 현금/상품권/외부 교환
- 유저 간 포인트 거래
- 실시간 베팅 구조

## 4) User Stories
- 사용자로서, 짧은 시간 내에 AI와 승부를 보고 싶다.
- 사용자로서, 내가 AI보다 잘했는지 확인하고 싶다.
- 사용자로서, 승부 결과를 바로 알림으로 받고 싶다.
- 사용자로서, 포인트/연승으로 성장감을 느끼고 싶다.

## 5) Non-Functional Requirements
- 성능: 1초 이내 UI 반응, 결과 API < 500ms 목표
- 보안: 인증/JWT, 입력값 검증, rate limit
- 확장성: 세션당 AI 1회, 캐시 사용
- 규제 리스크: 비현금 보상, 명확한 면책 문구
